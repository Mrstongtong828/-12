# 本地评分器说明

> `local_scorer.py` v3 — 《数据库敏感字段自动识别与安全管控系统》本地评估工具
>
> 对齐 README §6 官方评分公式，用于在提交靶机前完成本地验证与调优。

---

## 一、概述

本地评分器通过对比 `output/upload.csv`（预测结果）与 `tests/example.csv`（抽样答案）来估算靶机得分。核心设计原则：

- **公式严格对齐官方**：四项指标（分桶 F1、加权 F1、等级准确率、综合识别得分）与靶机一致
- **4 元组 Scope 过滤**：避免"同 record 未被抽样的 type"被错计为 FP，贴近靶机完整答案集的召回逻辑
- **多维诊断输出**：value-only F1、form 标错计数、天花板估算，帮助定位优化方向

---

## 二、官方评分公式

### 2.1 ① 分桶 F1（按 `data_form`）

对 6 种数据形态分别计算 F1：

```
precision = TP / (TP + FP)
recall    = TP / (TP + FN)
F1_form   = 2 × precision × recall / (precision + recall)
```

### 2.2 ② 加权 F1

```
加权 F1 = Σ (weight_form × F1_form)
```

各形态官方权重：

| 形态 | 权重 |
|------|------|
| `structured` | 10% |
| `semi_structured` | 20% |
| `encoded` | 20% |
| `unstructured_text` | 25% |
| `binary_blob` | 20% |
| `db_object` | 5% |

**注意**：`unstructured_text` 权重最高（25%），是优化重点。

### 2.3 ③ 等级准确率

```
等级准确率 = 等级正确的 TP 数 / 答案总行数
```

分母是**答案总行数**（含 FN），而非 TP 数，漏报会同时扣 recall 和等级准确率分（双重惩罚）。

### 2.4 ④ 综合识别得分（靶机返回值）

```
综合识别得分 = 加权F1 × 0.70 + 等级准确率 × 0.30
```

## 三、Scope 过滤机制

### 3.1 问题背景

`example.csv` 约为靶机完整答案的 1% 抽样（~566 条记录）。若直接在**全量预测**中计 FP，会把"同一行中靶机完整答案包含但 example 未抽到的 type"错算成误报，导致精确率虚低，误导向收紧正则的错误方向。

### 3.2 4 元组 Scope（默认）

过滤键：`(db_name, table_name, record_id, sensitive_type)`

只对 example 中出现的 `(db_name, table_name, record_id, sensitive_type)` 组合内的预测行参与评分，其余行静默丢弃（不计 FP）。

**数据验证**：example.csv 的 566 条记录中 552 条只出现 1 种 type，强证明同 record 的其他 type 大概率是靶机答案中的真实 TP。

### 3.3 3 元组 Scope（`--strict-scope`）

过滤键：`(db_name, table_name, record_id)`，回退 v1 行为，可用于对比验证。

## 四、CLI 用法

### 4.1 基本运行

```bash
# 默认：answer=tests/example.csv，pred=output/upload.csv
python local_scorer.py

# 指定路径
python local_scorer.py --answer tests/example.csv --pred output/upload.csv
```

### 4.2 差异分析

```bash
# 打印全部 FP/FN 明细（默认每类最多 30 条）
python local_scorer.py --diff

# 只看某个形态的 diff
python local_scorer.py --diff --form encoded

# 调整每类打印条数
python local_scorer.py --diff --limit 50

# 额外打印"值对但 form 标错"的条目（双重扣分项）
python local_scorer.py --diff --form-mismatch
```

### 4.3 其他选项

```bash
# 激进归一化（NFKC + casefold）再比对；默认只做 strip
python local_scorer.py --normalize

# 退回 v1 的 3 元组 scope（调试对比用）
python local_scorer.py --strict-scope
```

## 五、输出指标解读

### 5.1 [A] 分桶 F1

```
形态                F1     TP    FP    FN   权重
structured        0.920   230    18    22   10%  ████████████████████
unstructured_text 0.710   142    55    60   25%  ██████████████░░░░░░
...
```

关注 FP 和 FN 数量较大的行，对应优化优先级最高。

### 5.2 [B] 加权 F1 天花板

```
[天花板] 本地完美预测最多 0.9500（达成率 87.3%）
         抽样未覆盖的 form: ['db_object']
         对应综合识别得分本地天花板: 0.9650
```

若 `example.csv` 未抽到某 form 的样本，该 form 本地永远得 0，加权 F1 天花板 < 1.0。这是**抽样偏差**导致，不代表系统漏召，不应为此收紧该 form 的识别逻辑。

### 5.3 [C] 等级准确率

```
= 正确等级的TP数 / 答案总行数 = 312 / 400 = 0.7800
诊断：TP-only 等级准确率 = 312 / 350 = 0.8914
```

两个数值差距大说明**漏报（FN）多**，提升 recall 可同时改善等级准确率。

### 5.4 [D] 综合识别得分

对应靶机返回值。一般本地分比靶机分低 2–5 个百分点（抽样覆盖不完整）。

### 5.5 [E] Value-only F1（诊断用）

忽略 `data_form`，只看值是否找对。若 value F1 远高于加权 F1，说明**form 标错**（dispatcher 路由问题）是主要丢分点：修一次 dispatcher 即可同时消灭 FP 和 FN（双倍收益）。

### 5.6 Form 标错告警

```
[!] 值找对但 data_form 标错: 15 条
    官方分桶会把这些条目计为 FP+FN（双重扣分）。
    修 dispatcher 路由逻辑能一次吃下两倍收益。
```

优先级最高的修复项，每修 1 条等效提升 2 条 F1 贡献。

## 六、局限性与注意事项

| 局限 | 说明 |
|------|------|
| **抽样偏差** | example.csv 约 1% 抽样，本地分仅反映抽到部分。靶机用完整答案集，实际综合分通常高于本地估算 |
| **FP 虚高** | 若 value_fp 中大量合理敏感值，多为抽样遗漏的真实 TP，**不要为消灭 FP 收紧正则**，会压低靶机 recall |
| **db_object 天花板** | 若 example.csv 无 db_object 样本，本地该桶永远 0 分，无需在意 |
| **等级分不独立** | 等级准确率分母含 FN，先提 recall 再调等级映射，ROI 更高 |
| **form 权重不均** | unstructured_text(25%) + binary_blob(20%) + encoded(20%) 合计 65%，优化这三类效益最大 |

## 七、快速优化决策树

```
综合识别得分 偏低？
    │
    ├─ value_f1 >> 加权F1？
    │       └─ 是 → form 标错为主因 → 优先修 dispatcher 路由
    │
    ├─ 某 form 的 FN 多？
    │       └─ 是 → 该 form 召回不足 → 放宽对应正则/阈值
    │
    ├─ 某 form 的 FP 多？
    │       └─ 若为合理敏感值 → 很可能是抽样遗漏，不动
    │          若为噪声       → 加强校验/黑名单
    │
    └─ 等级准确率低？
            └─ 先看 FN 数量
               FN 多 → 提 recall 优先
               FN 少 → 查 sensitive_type→level 映射表
```
