# 作品简介

**作品名称**：数据库敏感字段自动识别与安全管控系统

**参赛赛事**：2026 年广东省大学生计算机设计大赛（网络与数据安全挑战赛）本科组

---

## 一、作品概述

本作品是一套面向 MySQL / PostgreSQL 数据库的**离线敏感数据穿透识别、分级分类、脱敏方案与权限管控一体化系统**。系统无需网络连接，可在赛题规定的 Intel i5-10400 / 16GB RAM / Windows 11 环境下，30 分钟内完成 4 个数据库的全量扫描，自动输出符合评分格式要求的 `upload.csv`。

---

## 二、核心功能

### 2.1 六种数据形态全覆盖识别

| 形态 | 处理方式 |
|------|---------|
| 结构化字段 | 字段名语义字典 + 20 种正则 + 校验算法 |
| 半结构化（JSON/XML） | 递归展开叶子节点，逐值扫描 |
| 编码变形（Base64/Hex/URL/Unicode） | 5 轮递归解码链，带四重熔断 |
| 非结构化自然语言文本 | 正则组合 + 百家姓锚点中文姓名识别 |
| 二进制图片/文档（BLOB） | PaddleOCR v4（CPU）+ PyMuPDF/python-docx/openpyxl + 多路径纠错 |
| 数据库对象（存储过程/视图） | 扫描 SQL 定义体中的硬编码敏感信息 |

### 2.2 20 种敏感类型识别

覆盖 ID_CARD、BANK_CARD、PHONE_NUMBER、CHINESE_NAME、ADDRESS、EMAIL、PASSPORT、MILITARY_ID、VIN_CODE、LICENSE_PLATE、USCC、BUSINESS_LICENSE_NO、SOCIAL_SECURITY_NO、HOUSING_FUND_NO、MEDICAL_INSURANCE_NO、MEDICAL_RECORD_NO、GPS_COORDINATE、IP_ADDRESS、MAC_ADDRESS、PASSWORD_OR_SECRET。

内置 GB 11643 身份证校验位、Luhn 银行卡校验、USCC 字符集校验，大幅降低误报率。

### 2.3 四级敏感分类

对齐《数据安全法》与 GB/T 35273-2020：
- **L4 高度敏感**：身份证、银行卡、护照、军官证、密码密钥
- **L3 敏感**：姓名、手机、邮箱、地址、车牌、各类证件号
- **L2 内部**：GPS 坐标、IP 地址、MAC 地址

### 2.4 脱敏工具

20 种敏感类型各有专属脱敏函数（部分掩码、哈希脱敏、位保留等），支持批量对比输出 `masked_report.csv`。

### 2.5 权限管控（RBAC）

4 角色（admin / auditor / operator / user）× 权限矩阵，配置文件 `permissions.yaml` 定义动态脱敏规则与数据导出审批流程。

## 三、技术亮点

1. **子进程化 OCR**：PaddleOCR 运行于独立子进程池，崩溃不影响主进程；行级超时 25s，单表熔断 540s，全程不中断扫描。

2. **多路径 OCR 扫描**：原始文本 / 数字纠错 / 汉字纠错 / 双修 / 去空格，共 5 路并行去重，极限提升图片内敏感信息召回率。

3. **校验位反馈枚举**：对 OCR 产出的数字串做 digit→digit 混淆 BFS，1-edit 内修复 OCR 常见 0/6、1/7 混淆，用 GB11643 / Luhn 校验验证后采纳。

4. **高精度中文姓名识别**：百家姓单姓 + 复姓词典 + 黑名单 + 行政区名守卫 + 动词后缀过滤，多重机制联防误报。

5. **离线运行**：无任何网络请求，所有模型（PaddleOCR ch_PP-OCRv4）预置于本地，严格符合赛题断网要求。
